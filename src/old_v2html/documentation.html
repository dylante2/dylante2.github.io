Data Types

There are four Data Types in Milk Lang. They are:

  • Booleans: Values that indicate true or false
  • Numbers: Any numerical representation that you can think of. 1234 and 12.34 fall under the same roof in Milk Lang.
  • Strings: Any letter or word falls under the String Data Type. They are declared by "double quotes."
  • Nil: Indicates nothing. A zero. Null.

Statements

Statements produce an effect. The print statement is an example. You can pack a series of statements into brackets, {}.

{ print"One statement."; print"Two statements."; }
Variables

Variables store data. Like other languages, use = to initialize variables. Omitting the initializer defaults to nil. Because Milk is dynamically-typed there is only one key word for defining variables, var.

var hello = "World"; var alsoAVar;

Users can access and assign the values of variables by refering its name.

var breakfast = "bagels"; print breakfast; // "bagels". breakfast = "beignets"; print breakfast; // "beignets".
Control Flow and Loops

Without the ability to skip code or execute code more than once, many languages would become useless. In Milk, the if, while, and for loop allow users to make powerful programs.

The if() statement executes one of two statemnts based on some condition

if(condition){ print "correct"; } else{ print "wrong"; }

The while() loop executes the body repeatedly until the conditional expression is no longer true.

var a = 1; while (a < 10) { print a; a = a + 1; }

The for() loop is a more "elegant" way of creating a while loop.

for (var a = 1; a < 10; a = a + 1) { print a; }
Functions

A function is a named section of code that performs a certain task. In Milk they can be run with or without arguments and look just like Java's function calls.

doThing() build(brick, clay)

To declare a function, use the keyword ming...LOL.

// No I'm not joking. ming printSum(a,b) { print a + b; }

In Milk, functions are first class, meaning they work just like real values. Imagine function pointers in C.

ming addPair(a, b) { return a + b; } br>ming identity(a) { return a; } print identity(addPair)(1, 2); // Prints "3".

Functions can be declared within functions. It's a very cool feature, shout-out to Bob Nystrom.

ming returnFunction() { var outside = "outside"; ming inner() { print outside; } return inner; } var woah = returnFunction(); fn();
Classes

Milk is an OOP language

class Breakfast { cook() { print "Eggs a-fryin'!"; } serve(who) { print "Enjoy your breakfast, " + who + "."; } } // Store it in variables. var someVariable = Breakfast; // Pass it to functions. someFunction(Breakfast); // Print it print someVariable; // "Breakfast instance"

For Instances and Initialization, use init() to create a construction-sorta-thing.

class Breakfast { init(meat, bread) { this.meat = meat; this.bread = bread; } } // ... var baconAndToast = Breakfast("bacon", "toast"); baconAndToast.serve("Dear Reader");

For Inheritance, use subClass < superClass

class Brunch < Breakfast { drink() { print "How about a Blood Mary?"; } }

In the above example Brunch is the subclass and thus is able to use any of Breakfast's methods. But when Brunch calls drink() the output is different than what Breakfast would output.
